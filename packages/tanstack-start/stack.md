# TanStack Start — Stack Reference

This document describes the standard forge markers and file conventions used
by the `tanstack-start` package and all pattern packs that target it.

---

## Standard Forge Markers

Forge markers are comment anchors placed in project files. When `tsx pattern run`
executes, it finds each marker and injects the rendered line immediately after it.
Injection is idempotent — the runner checks if the line is already present.

### `// [tsx:schemas]`

**File:** `db/schema/index.ts`
**Purpose:** Re-export every Drizzle table schema module.

```ts
// db/schema/index.ts
// [tsx:schemas]
export * from './users';    // ← injected by add-schema generator
export * from './sessions'; // ← injected by add-schema generator
```

Generated by: `tsx generate add:schema <name>` (via the `markers` field in `add-schema.json`)

---

### `// [tsx:server-fns]`

**File:** `src/server-fns/index.ts`
**Purpose:** Re-export every server-function module.

```ts
// src/server-fns/index.ts
// [tsx:server-fns]
export * from './auth';  // ← injected by add-server-fn generator
export * from './todo';  // ← injected by add-server-fn generator
```

Generated by: `tsx generate add:server-fn <name>` (via the `markers` field in `add-server-fn.json`)

---

### `// [tsx:providers]`

**File:** `src/routes/__root.tsx` (inside the `Providers` component)
**Purpose:** Mount React context providers without editing the root route manually.

```tsx
// src/routes/__root.tsx
function Providers({ children }: { children: React.ReactNode }) {
  return (
    <QueryClientProvider client={queryClient}>
      {/* [tsx:providers] */}
      {children}
    </QueryClientProvider>
  );
}
```

---

### `// [tsx:middleware]`

**File:** `src/middleware.ts`
**Purpose:** Register TanStack Start middleware in order.

```ts
// src/middleware.ts
import { createMiddleware } from '@tanstack/start';
// [tsx:middleware]

export const loggingMiddleware = createMiddleware().server(async ({ next, request }) => {
  console.log(`[${request.method}] ${request.url}`);
  return next();
});
```

## Marker Format by Language

| Language     | Marker syntax              |
|--------------|----------------------------|
| TypeScript   | `// [tsx:<name>]`          |
| Rust         | `// [tsx:<name>]`          |
| Vue SFC      | `<!-- [tsx:<name>] -->`    |
| TOML         | `# [tsx:<name>]`           |

## File Layout

```
src/
├── db/
│   └── schema/
│       ├── index.ts         ← // [tsx:schemas]
│       ├── users.ts
│       └── sessions.ts
├── server-fns/
│   ├── index.ts             ← // [tsx:server-fns]
│   ├── auth.ts
│   └── todo.ts
├── routes/
│   ├── __root.tsx           ← // [tsx:providers]
│   └── _protected/
│       └── ...
└── middleware.ts             ← // [tsx:middleware]
```

## Pattern Pack Integration

Any pattern pack targeting `tanstack-start` should declare markers in its `pack.json`:

```json
{
  "markers": [
    {
      "marker": "// [tsx:schemas]",
      "file": "db/schema/index.ts",
      "insert": "export * from './{{ name | snake_case }}';"
    },
    {
      "marker": "// [tsx:server-fns]",
      "file": "src/server-fns/index.ts",
      "insert": "export * from './{{ name | snake_case }}';"
    }
  ]
}
```

`tsx pattern lint` will warn if the marker target files do not exist in the project.
