# tsx Examples

Each sub-directory is a **frozen snapshot** of files generated by the `tsx` CLI using real forge templates. They are reference implementations — not live projects — but every line of code matches what the generators actually produce.

## Examples

| Example | Packages | Description |
|---------|----------|-------------|
| [basic-crud](./basic-crud/) | tanstack-start, drizzle-pg | Products CRUD with Drizzle ORM |
| [with-auth](./with-auth/) | + better-auth | Adds email/OAuth authentication |
| [with-shadcn](./with-shadcn/) | + shadcn | shadcn/ui form & data-table components |
| [full-saas](./full-saas/) | all four | Multi-entity SaaS starter (orgs, users, posts) |

## Regenerating an Example

Each example contains a `scripts/generate.sh` that documents the exact `tsx` commands used.

```bash
# Install the CLI
cargo install tsx

# Regenerate basic-crud from scratch
cd examples/basic-crud
bash scripts/generate.sh
```

## Notes

- Files require a running PostgreSQL database + `.env` with `DATABASE_URL` to actually execute
- The `.tsx/stack.json` shows which packages are active for each example
- See [registry.tsx.dev](https://registry.tsx.dev) for full package documentation
